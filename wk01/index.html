1.4 Common Types of Networks


1.4.1 Networks of Many Sizes

        Small Home - connects a few devices to the internet 
        Small Office / Home Office - computer within home to connect to corporate office
        Medium to Large Network - many locations with hundreds of interconnected computers
        World Wide Network - hundreds of millions of computers; ie: the internet 

1.4.2 LANs and WANs

        PAN (Personal Area Network) - bluetooth connected personal devices used on your person or in a small area 

        LAN (Local Area Network) - connects end devices in a limited area
                                   ie: single floor of office building, \
                                   ie02: a lab at school 

        CAN (Campus Area Network)  - multiple buildings and networks all joined into a larger network

        MAN (Multiple Area Network) - ie: police or fire department; many different networks connected through a city

        WAN (Wide Area Network)  - interconnects LANs over wider geographic areas
                                     ie: large company, google 

1.4.3 the Internet

        a worldwide collection of interconnected LANs and WANs 

        LANs are connected to each other using WANs

        3 main groups help keep internet structure: 
            - IETF
            - ICANN
            - IAB

1.4.4 Intranets and Extranets

        Intranet = internal business network
        Extranet = Suppliers, Customers, Collaborators; offers secure access to business
        Internet = worldwide access -- capital I = world wide web

1.6 Reliable Networks


1.6.1 Network Architecture

        technologies that support the infrastructure 

        4 Characteristics of a Reliable Network:
          - Fault Tolerance
          - Scalability
          - Quaility of Service
          - Security

1.6.2 Fault Tolerance

        limit the impact of failure by having multiple paths 

        provide redundancy by packet switching: 
            - packet switching splits traffic into packets that are routed over a network
            - each packet can (theoretically) take a different path and end at the correct destination 

        packet switched is the opposite of circuit-switched network    

1.6.3 Scalability

        allows for expansion and quickly updating needs without impacting performance

1.6.4 Quality of Service

        router manages flow of data and voice traffic to keep it from being jittery or lagging

1.6.5 Network Security

     SSH is more secure than Telnet, but it is hackable
        2 main types of security: 
            Network Infrastructure Security-
                - Physical Security of network devices
                - Preventing unauthorized access to devices and secure areas
            Information Security:
                - Protection of data while transmitted over the network
        3 goals of network security:
            - Confidentiality - only intended recipient can read data 
            - Integrity       - data is unaltered and matches the data from the source 
            - Availability    - assurance of timely and reliable access to data

2.2.1 Primary Command Modes

          limited number of basic monitoring commands
       
       Priviliged EXEC Mode - ex: Switch #  -- has '#' symbol as prompt

2.2.2 Configuration Mode and Subconfiguration Mode

        Golbal Configuration Mode - ex: Switch(config)#

        Line Configuration Mode   - ex: Switch(config-line)#
             used to configure Console, SSH, Telnet, or AUX access
             
        Interface Configuration   - ex: Switch(config-if)#

2.3.5 Hot Keys and Shortcuts

        TAB        - Completes partial command entry
        Backspace  - Erases the character to the left of the cursor
        Ctrl + D   - Erases the character at the cursor
        Ctrl + K   - Erases all characters from the cursor to the end of the command line
        Esc D      - Erases all characters from the cursor to the end of the word
        Ctrl + U   - Erases all characters from the cursor back to the beginning of the command line; also Ctrl + X
        Ctrl + W   - Erases the word to the left of the cursor
        Ctrl + A   - Moves the cursor to the beginning of the line
        Ctrl + B   - Move the cursor 1 character left; also Left Arrow Key
        Esc B      - Moves the cursor back 1 word
        Esc F      - Moves the cursor forward 1 word
        Ctrl + F   - Moves the cursor 1 character to the right; also Right Arrow Key
        Ctrl + E   - Moves the cursor to the end of the command line
        Ctrl + P   - Recalls previous command in history buffer, beginning with most recent; also Up Arrow
        Ctrl + N   - Goes to the next line in the history buffer; also Down Arrow
        Ctrl + R   - Redisplays system prompt and command line after a console message is received; also Ctrl + I or Ctrl + L
        Ctrl + C   - when in 'configuration mode': Ends configuration mode; when in 'setup mode': Aborts back to command prompt 
        Ctrl + Z   - when in any 'configuration mode': Ends configuration mode and returns to 'privileged EXEC mode'
        Ctrl+Shift+6 - All purpose break sequence used to about DNS lookups, pings, traceroutes, etc...
